# 项目定义
本项目是一个中医问诊辅助系统

# 项目目标
- 帮助用户描述症状
- 引导用户补充关键信息
- 基于中医知识库提供参考解释
- 输出结构化问诊摘要
- 对高风险情况进行提醒

# 不做的事情
- 不输出确定性诊断结论
- 不替代医生
- 不输出高风险用药建议
- 不承诺治疗效果

# 项目最终形态
一个支持多轮对话、知识检索、风险提示、结构化摘要的 LLM 应用
# 目标用户
- 普通用户：希望表达自己的不适并获得参考性问诊引导
- 演示用户/面试官：查看完整的大模型开发流程
- 开发者自己：学习 RAG、微调、评估、部署

# 用户痛点
- 不知道如何准确描述症状
- 不知道医生问诊通常会追问什么
- 中医术语理解困难
- 缺少结构化的病情整理
# 输入
- 用户自然语言描述，例如：最近胃胀，饭后更明显，睡眠一般

# 输出
- 已抽取的症状信息
- 需要继续追问的问题
- 基于知识库的参考解释
- 风险提示
- 问诊摘要

# 最小闭环
用户输入症状 -> 系统抽取关键信息 -> 发现信息不足 -> 继续追问 ->
检索知识 -> 给出参考回复 -> 输出结构化摘要
# 安全规则
- 系统不做确定性诊断
- 系统不替代线下就医
- 遇到高风险症状优先提示及时就医
- 不提供处方建议
- 回答必须带“仅供参考”的边界说明
# 高风险症状示例
- 持续胸痛
- 呼吸困难
- 意识模糊
- 高热不退
- 突发剧烈腹痛
- 呕血/便血
- 严重外伤
# 阶段性成功标准

    ## 阶段0成功标准
    - 能清晰定义项目边界
    - 能列出输入输出和风险规则
    - 能定义后续要做的模块

    ## 项目整体成功标准
    - 能进行多轮问诊
    - 能输出结构化摘要
    - 能进行知识检索增强
    - 能对高风险症状做提醒
    - 能完成基础评估和部署
