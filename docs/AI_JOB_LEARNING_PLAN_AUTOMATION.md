# AI 求职与学习周计划自动化

本流程用于把求职、项目、学习、考试复习和内容发布串成一个可复盘的 Obsidian 工作流。

当前核心判断：最重要的不是“学更多东西”，而是把学习、项目、内容、简历串起来。每个任务都必须回答至少一个问题：

- 能不能写进简历？
- 能不能服务多 Agent 协作工作台？
- 能不能形成面试表达？
- 能不能复用成简历、项目说明或面试表达？
- 是不是当前考试前必须做？

如果五个问题都答不上来，就不要安排。

## 固定模块

每周计划必须包含 7 个模块：

- A. 简历项目整理与工程化
- B. 多 Agent 协作工作台推进
- C. AI 前沿 / 热门架构学习
- D. 长期技术栈补充
- E. 算法 Hot100
- F. 内容发布（暂停）
- G. 考试复习与缓冲

每周必须明确本周主目标、模块优先级、交付物、最低完成标准、可砍任务、不可砍任务、每日动态计划规则、每日复盘和周计划调整规则。

## 默认优先级

- P0：简历项目整理与工程化。优先完成可写进简历、可展示、可面试追问的项目材料。
- P1：缓冲与复盘。考试暂时结束，不参与每日任务加权。
- P2：算法 Hot100。每天至少 5 分；easy=1，medium=2，hard=3；不能连续两天低于目标。
- P3：AI 前沿 / 热门架构学习。每天 1 个主题，必须服务简历项目、工作台或面试表达。
- P4：长期技术栈补充。每天约 1 小时，偏大模型应用开发和 infra。
- P5：内容发布暂时停止。暂停期间不生成小红书发布、草稿或选题任务。

## 考试规则

- 考试暂时结束，不生成考试复习 P0 / P1。
- 后续如重新启用考试计划，需要明确考试日期和复习窗口。

## 动态调整规则

- 简历项目连续 2 天没有实质推进时，第三天必须把简历项目设为 P0。
- 算法按积分制保底：easy=1，medium=2，hard=3；每天至少 5 分；连续两天低于目标时，第三天必须补足并做错题复盘。
- 小红书计划暂停，不参与每日任务加权。
- 每日 AI 前沿学习必须映射到简历项目、多 Agent 工作台或面试表达之一。
- 长期技术栈学习必须绑定 API、数据库、任务队列、并发、网关、记忆、eval、tracing、deployment 或 security 之一，不能只看教程。
- 昨日完成度 < 60% 或主观精力 <= 5/10 时，今日自动降载。
- 昨日完成度 >= 85% 且精力 >= 7/10 时，今日可以增加 1 个项目推进任务，但不能增加超过 1 个新主题。
- 每周最多设置 1 个主项目主线，除非项目阻塞，不要每天切换核心目标。
- 每周最后一天必须输出周复盘。

## 阶段计划

### Week 1：2026-06-13 - 2026-06-19

定位：考试周 + 简历项目启动周。

主目标：

- 完成第一轮简历项目盘点，明确 2-3 个主项目包装方向。
- 考试暂时结束，不纳入本周交付。
- 小红书计划暂停，不纳入本周交付。
- 保持每日 AI 前沿学习，但不做过重 demo。
- 保持算法 Hot100 连续性。

不可砍：简历项目主线、算法每日 5 分、每日复盘。

可砍：复杂 demo、非必要新技术深挖、小红书发布与选题、多 Agent 工作台重型开发。

### Week 2：2026-06-20 - 2026-06-28

定位：高强度项目冲刺周。

主目标：

- 把简历项目推进到可写进简历的完整版本。
- 多 Agent 协作工作台完成 MVP 设计并开始核心骨架实现。
- 考试暂时结束，不纳入本周交付。
- 保持算法和前沿学习不断档。
- 把每日学习内容反哺项目和简历。

不可砍：简历项目最终包装、多 Agent 工作台 MVP 设计、算法每日 5 分、每日复盘。

可砍：小红书发布与选题、非核心技术栈扩展、泛 AI 新闻、超出 MVP 的工作台功能。

### Week 3：2026-06-29 - 2026-07-05

定位：实习低强度过渡周。

主目标：

- 考试暂时结束，切换为公司实习期间的低强度维护节奏。
- 保持算法和 AI 前沿不断档，但降低每日负担。
- 简历项目进入打磨和投递准备阶段。
- 多 Agent 工作台进入小步迭代。

不可砍：每日复盘、简历项目打磨、算法每日 5 分、每日轻量 AI 前沿学习。

可砍：大规模新功能开发、重型技术栈 demo、小红书发布与选题、超过 1 小时的非必要探索。

## 每日复盘输入

每天生成日计划前，必须先读取昨日 Daily Note，并尽量提取：

- 昨日计划任务列表
- 实际完成任务
- 未完成任务
- 每项任务耗时
- 昨日算法完成数量
- 昨日前沿学习主题
- 昨日技术栈学习内容
- 昨日简历 / 项目推进情况
- 昨日内容发布状态（暂停期间可不填）
- 昨日缓冲事项
- 昨日主观状态：精力、专注度、焦虑程度、睡眠情况

复盘输出必须包含完成度评分、昨日评价、未完成任务处理、本周剩余计划调整和今日计划生成依据。

## 每日计划输出

每日计划必须写入 `01_Daily/YYYY-MM-DD.md` 的 `PERSONAL_WORKSTATION_DYNAMIC_PLAN` 标记区块，并包含：

- 昨日复盘
- 今日动态调整
- 今日 P0 任务
- 今日 P1 任务
- 今日最低保底任务
- 今日可选加码任务
- 今日交付物
- 晚间复盘输入
- 未完成任务处理区

## 周计划输出

周计划写入 `05_Reviews/Weekly/YYYY-MM-DD_YYYY-MM-DD_Plan.md`，格式必须包含：

- 本周定位
- 本周主目标
- 本周优先级
- 七个模块计划
- 本周不可砍任务
- 本周可砍任务
- 每日动态计划规则
- 风险提醒
- 本周成功标准

每日运行会同步更新周计划中的 `PERSONAL_WORKSTATION_WEEKLY_ADJUSTMENT` 标记区块，记录当日动态权重。

算法计分规则：

- 每天最低 5 分。
- easy = 1 分。
- medium = 2 分。
- hard = 3 分。
- 复盘时优先记录每题难度、题型、错因和模板。

日报边界：

- Daily Note 只保留人工可读的目标、动态计划、执行记录、复盘和明日下一步。
- 不再写入审核状态区块。
- 不再把 `PERSONAL_WORKSTATION_DAILY_SYNC` 自动汇总写入日报；机器汇总写入 `99_System/Daily_Sync/YYYY-MM-DD.md`。
- 不再生成“今日不做什么”区块。

## 本地命令

生成或更新今日动态计划：

```powershell
python -B scripts\personal_workstation\workstation.py --config configs\personal_workstation.local.json adjust-plan
```

生成指定日期计划：

```powershell
python -B scripts\personal_workstation\workstation.py --config configs\personal_workstation.local.json adjust-plan --date 2026-06-14
```

只生成本周计划：

```powershell
python -B scripts\personal_workstation\workstation.py --config configs\personal_workstation.local.json adjust-plan --date 2026-06-14 --weekly-only
```
