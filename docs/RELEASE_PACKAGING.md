# Release Packaging

Generated: 2026-06-18

## P3.3 Goal

P3.3 packages the current project as a local reproducible prototype / local
engineering release candidate. In manifest terms:
`local reproducible prototype / local engineering release candidate`.

It gives a new local reviewer one place to find runtime modes, environment
variables, validation commands, generated artifacts, and release boundaries.

P3.3 is not:

- production medical product
- diagnosis system
- prescription system
- treatment planning system
- multi-user platform
- hospital deployment
- Web UI
- complex Docker deployment
- CI/CD platform

## Project Position

TCM-Assistant is currently a local engineering prototype for collecting inquiry
information and validating API/runtime behavior. It preserves the P1/P2 API
response body contract and the existing SQLite schema.

Safety boundary phrases that must remain true:

- 不诊断
- 不开方
- 不替代医生
- 高风险提示线下就医

## Environment Preparation

Use Python 3.11 or a compatible local Python environment from the project root.
Create an isolated environment if desired, then install dependencies:

```bash
python -m pip install -r requirements.txt
```

The package is meant to run locally from the project root. Documentation and
manifest files use relative paths such as `.runtime/`, `artifacts/`, `docs/`,
`scripts/`, and `tests/`.

## Environment Variables

Start from `.env.example` if you need a local `.env` file. The example contains
placeholders only:

```env
TCM_RUNTIME_MODE=local
TCM_API_DB_PATH=.runtime/tcm_assistant.sqlite3
TCM_RUNTIME_DIR=.runtime
TCM_ARTIFACTS_DIR=artifacts
TCM_ALLOW_REAL_LLM=false
TCM_LOG_LEVEL=INFO
TCM_REDACT_LOGS=true
TCM_CONFIG_STRICT=false
OPENAI_API_KEY=
```

`.env` is local-only and should not be committed to Git. By default,
`TCM_ALLOW_REAL_LLM=false`, so the local package does not require or enable a
real LLM key.

## Runtime Modes

`local` is the default mode for local development and uses
`.runtime/tcm_assistant.sqlite3`.

`test` is for unit and integration tests. Tests should use temporary database
paths or controlled fixtures.

`demo` is for controlled local demo scripts. Outputs must remain redacted.

`eval` is for case corpus, long-session, and gate evaluations. Eval outputs are
written as machine-readable artifacts.

## API Startup And Health

Start the local API:

```bash
uvicorn app.api.main:app --reload
```

Check health:

```bash
python scripts/run_api_demo.py
```

or call:

```bash
curl http://127.0.0.1:8000/health
```

The additive `X-Request-ID` response header is supported by P3.2. Response body
schemas are not changed by P3.3.

## Reproducibility Commands

Recommended P3.3 validation commands:

```bash
python scripts/check_runtime_config.py
python scripts/check_observability.py
python scripts/check_release_packaging.py
python scripts/run_p1_gate.py
python scripts/run_p2_gate.py
python scripts/run_case_corpus_eval.py
python scripts/run_long_session_demo.py
python scripts/secret_scan.py
python -m unittest discover -s tests -p "test*.py"
git diff --check
```

Use the explicit unittest discovery command above for full local discovery.
Running only `python -m unittest discover` may discover 0 tests in this
repository layout, so it should not be treated as the full test result.

## Artifacts

`artifacts/p3_3_release_packaging.json` is the release manifest. It describes
the package contents, runtime modes, required commands, boundaries, and
non-goals.

`artifacts/p3_3_release_packaging_check.json` is generated by
`scripts/check_release_packaging.py` and records whether required docs,
artifacts, environment examples, and boundaries are valid.

Other gate artifacts are generated by P1/P2/P3.1/P3.2 scripts and may be
refreshed by the validation commands.

## Docs

`docs/RUNTIME_CONFIG.md` documents P3.1 runtime config.

`docs/OBSERVABILITY.md` documents P3.2 structured logging and redaction.

`docs/RELEASE_PACKAGING.md` documents P3.3 local reproducibility.

`docs/P3_3_RELEASE_PACKAGING_REPORT.md` records implementation and validation
results for this phase.

## Secret Policy

Real API keys, local `.env` files, and private runtime data are not included in
the release package. `.env.example` contains placeholders only. Structured logs,
gate subprocess output, and artifacts must redact secret-like values.

`.runtime/` is local generated state and ignored by Git. It is not part of the
release documentation set.

## Boundary

P3.3 does not add Web UI, users, permissions, ORM, MemoryManager, embeddings,
Tool Registry, multi-agent behavior, diagnosis output, prescription output, or
treatment-plan output.

P3.3 does not change the P1/P2 API response body contract and does not change
the SQLite schema.

## Known Limitations

This is a local reproducibility package, not an installer. It does not provide
service hosting, dashboards, API versioning, release-candidate policy, or
automated CI.
