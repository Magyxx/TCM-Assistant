## Gate Runner Behavior

The runner separates blocking hard checks from advisory soft checks:

- Hard failures set `hard_gate_status` to `failed` and return a non-zero exit
  code.
- Soft failures set `soft_report_status` to `caution` but do not fail the hard
  gate.
- The runner writes a bootstrap artifact before validating
  `artifacts/code_health_gate_baseline.json`, then writes the final artifact with
  all check results.
- `--skip-soft` can be used for fast hard-gate-only local verification.
- `--json` prints the machine-readable result to stdout after writing the
  artifact.
