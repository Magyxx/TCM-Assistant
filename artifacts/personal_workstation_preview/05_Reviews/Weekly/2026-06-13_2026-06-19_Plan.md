# 本周计划：2026.06.13 - 2026.06.19

## 1. 本周定位
考试周 + 简历项目启动周

## 2. 本周主目标
- 完成第一轮简历项目盘点，明确 2-3 个主项目的包装方向。
- 为 6 月 18 日考试留出复习时间。
- 小红书计划暂停，不纳入本周交付。
- 保持每日 AI 前沿 / 热门架构学习，但不做过重 demo。
- 保持算法 Hot100 连续性。

## 3. 本周优先级
- P0：简历项目整理与工程化；考试前 3 天自动切换为考试复习 P0。
- P1：考试复习与缓冲。
- P2：算法 Hot100 最低连续性。
- P3：AI 前沿 / 热门架构学习，必须能服务简历、工作台或面试。
- P4：长期技术栈补充。

## 4. 本周模块计划

### A. 简历项目整理与工程化
- 本周目标：完成第一轮简历项目盘点，并把 toB 多源订单系统、字节短剧 / 多 Agent pipeline 作为主包装候选。
- 本周交付物：
  - 可写进简历的项目清单。
  - 2-3 个主项目筛选结果。
  - 每个主项目的背景、技术栈、架构、职责、难点、方案、量化结果、面试追问。
  - toB 多源订单智能采集与结构化处理系统的简历初稿。
  - 字节短剧 / 多 Agent pipeline 项目的第一版包装思路。
- 最低完成标准：至少完成项目清单、主项目筛选和 1 个主项目初稿。
- 可加码内容：补齐 2 个主项目的 STAR 讲述和面试追问。

### B. 多 Agent 协作工作台推进
- 本周目标：只做设计和模块拆分，不做重型开发。
- 本周交付物：
  - 工作台总体架构草图。
  - 核心模块列表。
  - MVP 功能范围。
  - 数据表初稿。
  - Agent Runtime / Memory / Gateway / Event / Artifact / Approval 模块边界。
- 最低完成标准：形成文字版架构边界和 MVP 范围。
- 可加码内容：补一个模块间数据流草图。

### C. AI 前沿 / 热门架构学习
- 本周目标：每天 1 个工程主题，考试前两天只做轻量笔记。
- 本周主题池：
  - Agent Harness / Agent Loop
  - MCP
  - LangGraph Stateful Runtime
  - Context Engineering
  - Agent Memory Architecture
  - Model Gateway
  - Tool Gateway
- 每日最低产出：每天输出 5-8 条可复用笔记。
- 如何复用到项目 / 面试：必须映射到简历项目、多 Agent 工作台或面试表达之一。

### D. 长期技术栈补充
- 本周目标：补齐 Python 工程化和后端 infra 基础。
- 本周技术栈：
  - Python typing / Pydantic -> API schema
  - FastAPI 基础接口设计 -> API
  - PostgreSQL 表设计 -> 数据库
  - Redis session / task state -> 状态管理
  - asyncio 并发基础 -> 并发
- 对应功能：API, API schema, 并发, 数据库, 状态管理
- 最低 demo / 代码产出：至少 3 个技术栈学习笔记和 1 个可复用 demo / 伪代码设计。

### E. 算法 Hot100
- 本周目标：本周 10-12 道，考试前可降到每天 1 道。
- 本周目标题数：10-12 道
- 本周重点题型：哈希, 双指针, 滑动窗口, 二叉树
- 错题复盘要求：不能连续两天为 0；周内整理错题和模板。

### F. 内容发布（暂停）
- 当前状态：暂停：暂不安排发布、草稿和选题，只保留已有素材。
- 发布节奏：暂停；不生成发布、草稿或选题任务。
- 最低完成标准：暂停期间不生成每日发布任务。
### G. 考试复习与缓冲
- 本周目标：6 月 18 日考试前自动提高复习权重。
- 考试日期：2026-06-18；2026-06-29
- 本周复习目标：6 月 18 日考试前自动提高复习权重。
- 考前调整规则：距离考试 <= 3 天，考试复习升为 P0；距离考试 <= 1 天，不安排重型开发。
- 最低完成标准：明确考试范围，整理重点知识点，完成至少一轮核心内容复习。

## 5. 本周不可砍任务
- 6 月 18 日考试复习。
- 简历项目主线。
- 算法 Hot100 最低连续性。
- 每日复盘。

## 6. 本周可砍任务
- 复杂 demo。
- 非必要的新技术深挖。
- 小红书发布、草稿和选题打磨。
- 多 Agent 工作台的重型代码开发。

## 7. 每日动态计划规则
- 每天先复盘昨日完成情况，再生成今日计划。
- 今日计划必须从本周计划派生，不能孤立加新主线。
- 距离考试 <= 3 天时，考试复习自动升为 P0。
- 简历项目连续 2 天无实质推进时，第三天简历项目必须升为 P0。
- 算法不能连续两天为 0；连续两天低于目标时，第三天至少安排 2 道并做错题复盘。
- 小红书计划暂停，不参与每日任务加权。
- 昨日完成度 < 60% 或精力 <= 5/10 时，今日自动降载。

## 8. 风险提醒
- 简历项目被泛学习稀释，导致没有可投递材料。
- 考试临近时仍安排重型开发，造成复习失控。
- 前沿学习开新坑，不能反哺项目、简历或面试。

## 9. 本周成功标准
- 本周不可砍任务全部有明确产出。
- 简历项目比本周开始时更可展示、可追问、可投递。
- 算法保持连续，且有错题 / 模板沉淀。
- 每日计划和复盘都落档到 Obsidian。
